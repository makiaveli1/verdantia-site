## Scoring guide

Score each use case from 1–5.

| Score | Value | Risk / control need | Ease |
|---:|---|---|---|
| 1 | Little benefit | Low risk / simple controls | Very hard |
| 3 | Useful improvement | Moderate risk / needs clear review | Possible with support |
| 5 | Strong business/team value | High risk / needs strong controls | Easy to test safely |

Important: **do not total the risk score as if higher is better**. Use it as a warning flag. Good first experiments are usually high value, easy enough to test, and low-to-manageable risk.

## Facilitator scoring note

A good first experiment usually has:

- clear owner
- low data sensitivity
- visible time/quality benefit
- simple review process
- no high-stakes automated decision
- easy rollback if the output is poor
